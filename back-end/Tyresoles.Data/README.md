# Tyresoles.Data

Business logic and data access layer. Uses **Tyresoles.Sql** for database access and **generated models** for strongly-typed entities.

## Generated models

Models are generated by the CLI and live under namespace **`Dataverse.{SchemaSource}`** (e.g. `Dataverse.NavLive` when `SchemaSource` is `"NavLive"` in Tyresoles.config.json).

Regenerate from the **back-end** folder:

```bash
tyresoles generate
# or
dotnet run --project Tyresoles.Sql.Cli -- generate
```

Generated file: `back-end/Generated/NavModels.g.cs`.

## Usage

1. Reference **Tyresoles.Data** from your API or app.
2. Register in DI: `services.AddScoped<IDataverseDataService, DataverseDataService>();` (and ensure Tyresoles.Sql is registered).
3. In business logic, use `IDataverseDataService.ForTenant("NavLive")` and query with generated types:

```csharp
using Tyresoles.Data;
using Tyresoles.Sql.Abstractions;
// Use the namespace matching your SchemaSource (e.g. Dataverse.NavLive)
using Dataverse.NavLive;

// In your service
using (var scope = _dataService.ForTenant("NavLive"))
{
    scope.WithCompany("Your Company Name"); // optional, for company-scoped tables
    var customers = await scope.Query<Customer>()
        .Where(c => c.Blocked == false)
        .OrderBy(c => c.No)
        .ToArrayAsync();
}
```

Put all domain and data-access business logic in this project; keep Tyresoles.Api thin (controllers + DI wiring).
